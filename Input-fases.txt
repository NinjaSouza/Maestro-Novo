# Se ausente ou inválido, o sistema usa composição básica do Input-simulador.txt
#
# ============================================================================
# FORMATO (obrigatório — 4 campos separados por espaço/tab):
#
#   CAMADA_ID   MATERIAL   MASSA_G   TEMP_K
#
#   • CAMADA_ID : nome da camada (deve coincidir com as camadas do simulador)
#                 Use CAMADA1, CAMADA2, etc. — sem espaço
#   • MATERIAL  : nome ou alias do material (ver tabela abaixo)
#                 Case-insensitive: UO2_20 = uo2_20
#   • MASSA_G   : massa do material nesta camada, em gramas (float)
#                 Decimais permitidos: 10.5, 0.001725
#   • TEMP_K    : temperatura inicial da camada, em Kelvin (float)
#                 Mínimo: 250 K | Máximo: 3500 K
#                 Temperatura ambiente = 300.0 K
#
# REGRA: Uma linha por par (camada, material).
#        Uma camada com 2 materiais = 2 linhas com mesmo CAMADA_ID.
#        As massas não precisam somar exatamente — o parser calcula as
#        frações mássicas automaticamente: fração_i = massa_i / soma_total.
#
# COMENTÁRIOS: qualquer texto após '#' é ignorado.
# LINHAS COM < 4 CAMPOS: ignoradas com warning no log.
# ============================================================================
